# ТЗ: сайт-визитка (портфолио)

## 1. Цель

Личный сайт AI-инженера: агрегирует статьи с LinkedIn и Medium в одном месте, показывает опыт/проекты, служит опорной точкой в позиционировании при поиске работы. Не лендинг с лидген-формой — резюме-сайт для рекрутеров/собеседующих.

## 2. Дизайн-направление

**Стиль:** минимализм, без лишних деталей. Явно отвергнут вариант "как индийские портфолио" — перегрузка рамочками/бейджами/пилюлями создаёт ощущение шума даже при сдержанной палитре. Рамка/бордер — только там, где реально что-то группирует (карточка чата), не как декоративный элемент по умолчанию.

**Нельзя:**
- шрифты с засечками
- эффекты на курсор мыши
- анимированный/неоднотонный фон (без градиентов, без частиц/starfield)
- тяжёлые картинки
- аниме/декоративные иллюстрации
- визуальная перегруженность деталями (боксы/пилюли/бейджи россыпью)

**Можно/нужно:**
- мало цветов, но чтобы сочетались (палитру и шрифты Константин пришлёт отдельно — сейчас не финализировано)
- моноширинный шрифт как утилитарный акцент (даты, теги, лейблы) — не весь сайт целиком

**Референсы (что взято, что отклонено):**
- `portfolio-nailheart.vercel.app` — ориентир по сдержанности/минимализму палитры (не по контенту — там аниме-элементы, это как раз то, чего быть не должно).
- `akkila.dev` — идея чат-виджета с ИИ, стримингового, отвечающего только на вопросы про проекты/резюме. Взято как фича, но НЕ в виде полноценного терминального интерфейса — секция внутри обычной страницы, чтобы рекрутер не терялся.
- `yoav.xyz` — идея игры-астероидов на фоне. Понравилась как приём, но отклонена — не в тему AI-инженера.
- `utkarshs-terminal.netlify.app` — сайт-визитка целиком в виде терминала. Отклонено — рекрутер может не разобраться, что к чему.
- `leerob.com/beliefs` — формат страницы "things I believe": короткие пункты, сгруппированные по темам, без длинных абзацев. Взят как формат для секции Beliefs.

**Прототип (концепт-макет, HTML/CSS/vanilla JS, без реального бэкенда):**
https://claude.ai/code/artifact/7f3397f5-0774-4557-8c7f-d30c4ce82a62
Показывает структуру, плотность блоков и упрощённый чат-виджет (после декоттера). Весь текстовый контент в макете — заглушки, не реальные факты.

## 3. Структура страницы (секции)

1. **Header/nav** — минимальный, моно-лейблы, фиксированный сверху.
2. **Hero** — имя, позиционирующая строка, один абзац лида. Без ряда пилюль/бейджей под ним.
3. **Ask panel** — чат про проекты/резюме. MVP: несколько кликабельных вопросов-чипов + область ответа со стриминг-эффектом. Без отдельной шапки-панели и без поля свободного ввода в первой версии (можно добавить позже, когда бэкенд будет готов).
4. **Experience** — годы / компания / роль / описание / стек одной строкой текстом (не пилюлями).
5. **Projects** — карточки: заголовок / одна строка описания / стек текстом. Разделитель — тонкая линия сверху карточки, не рамка вокруг всего грида.
6. **Writing** — агрегация статей: источник (Medium/LinkedIn) / заголовок-ссылка / дата.
7. **Beliefs** — формат leerob: 3 группы по темам, короткие пункты.
8. **Footer** — email / linkedin / github / medium.

## 4. Контент — через файлы, без CMS

- Experience, Projects, Beliefs — Markdown/MDX с frontmatter (Astro content collections).
- LinkedIn-статьи — YAML/JSON список `{title, url, date}`, правится руками при публикации новой статьи (у LinkedIn нет RSS).
- Medium-статьи — тянутся автоматически по RSS (`medium.com/feed/@username`) на этапе сборки, руками не трогать.

## 5. Стек

- **Astro** — статический сайт-генератор. Node.js нужен только на этапе сборки (`astro build`), на VPS Node не работает как процесс — только готовая статика.
- **Чат-бэкенд** — отдельный сервис на **FastAPI** (Python), пишется отдельно, **сейчас не делаем**. Часть про LLM-провайдера отложена; по личному правилу для пет-проектов — без Anthropic SDK (см. память `feedback_vibecoding_workflow`), кандидат по умолчанию — OpenAI (gpt-4o-mini), не финализировано.

## 6. Деплой

```
npm run build                          # → dist/, чистая статика
rsync -avz --delete dist/ user@vps:/var/www/site/
```

nginx: раздаёт `dist/` как статику, путь `/api/` зарезервирован под будущий чат-бэкенд (сейчас `return 503;`, потом `proxy_pass http://127.0.0.1:8000;`). TLS — `certbot --nginx -d <домен>`.

Автоматизация (не блокирует MVP, добавить позже): GitHub Actions — build + rsync по SSH на push в main.

## 7. Открытые вопросы

- Шрифты и цветовая палитра — пришлю отдельно.
- Домен — не выбран/не подтверждён (без него certbot не выпустит сертификат).
- LLM-провайдер для чат-бэкенда — не финализирован (см. §5).
- Реальный контент (компании, проекты, тексты статей, beliefs) — в макете только заглушки, нужно заменить перед деплоем.

## 8. Критерий готовности

Рабочий сайт на VPS с URL (статика без чата — этого достаточно для первой итерации). Чат-бэкенд — отдельная последующая итерация.
