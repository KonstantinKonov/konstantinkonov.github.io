# Сайт-визитка (портфолио) — текущее состояние

> Этот документ описывает **реальную реализацию**. Изначальное ТЗ разошлось с
> тем, что получилось (мультистраничность, два языка, тёмная/светлая тема,
> деплой на GitHub Pages вместо VPS) — ниже зафиксировано, как есть сейчас.
> Отклонения от первоначального замысла помечены **[изменилось]**.

## 1. Цель

Личный сайт AI-инженера: опыт, проекты, статьи, «beliefs». Опорная точка в
позиционировании при поиске работы, а не лендинг с лидген-формой.

## 2. Дизайн-направление

Минимализм, сдержанная монохромная палитра, моноширинный шрифт как утилитарный
акцент (навигация, даты, теги, чипы), тонкие линии-разделители вместо рамок.
Без засечек, без анимированного фона, без декоративных иллюстраций.

**[изменилось]** Добавлен переключатель **тёмной/светлой темы** (в исходном ТЗ
его не было). Тема сохраняется в `localStorage`, применяется до отрисовки
инлайн-скриптом в `<head>` (нет вспышки неправильной темы).

## 3. Структура — мультистраничная **[изменилось]**

Изначально задумывалась одна страница со секциями. Реально — несколько страниц
с общими шапкой/лейаутом и переходами между ними (Astro `ViewTransitions`):

- **Header** ([src/components/Header.astro](src/components/Header.astro)) —
  фиксированный сверху, моно-навигация (Home / projects / writing / beliefs),
  переключатель языка `RU | EN`, переключатель темы. Persist между переходами;
  локаль-зависимые ссылки пересчитываются на клиенте от текущего пути.
- **`/`** ([src/pages/index.astro](src/pages/index.astro)) — Hero + Ask-панель +
  Experience.
- **`/projects`** ([src/pages/projects.astro](src/pages/projects.astro)) —
  сетка карточек (заголовок / описание / стек текстом, опциональная превьюшка).
- **`/writing`** ([src/pages/writing.astro](src/pages/writing.astro)) — список
  статей (источник / заголовок-ссылка / дата).
- **`/beliefs`** ([src/pages/beliefs.astro](src/pages/beliefs.astro)) — формат
  leerob: группы по темам, короткие пункты.

## 4. Двуязычность (i18n) **[изменилось]**

Не было в исходном ТЗ. Английская версия — в корне, русская — под `/ru/`
(зеркало: [src/pages/ru/](src/pages/ru/)). Контент дублируется по языкам в
[src/content/pages/en/](src/content/pages/en/) и
[src/content/pages/ru/](src/content/pages/ru/). Переключатель языка в шапке
сохраняет текущую страницу при смене локали.

## 5. Ask-панель

Чат-виджет «спроси про меня» ([src/components/AskPanel.astro](src/components/AskPanel.astro)):
кликабельные чипы-вопросы + область ответа с эффектом печати (стриминг-каретка,
уважает `prefers-reduced-motion`).

**[изменилось]** В ТЗ поле свободного ввода откладывалось «до бэкенда» — по факту
поле **уже есть**. Пока работает как заглушка: сопоставляет ввод с пресетными
вопросами, иначе показывает fallback («ассистент ещё не запущен»). Реального
LLM-бэкенда нет.

## 6. Контент — Astro content collections **[частично изменилось]**

Одна коллекция `pages` ([src/content/config.ts](src/content/config.ts)): **один
Markdown-файл на страницу на язык**, а списки (`experience`, `ask`, `projects`,
`writing`, `beliefs`) лежат массивами во frontmatter. Это отличается от исходной
идеи «отдельный MD-файл на каждый item».

- **Writing** — массив во frontmatter (`title`, опц. `source`/`date`/`url`/
  `image`), правится руками. **[изменилось]** Автоподтяжки Medium по RSS нет.

> ⚠️ Весь текстовый контент сейчас — заглушки (lorem ipsum). Нужно заменить на
> реальные компании/проекты/статьи/beliefs перед публичным анонсом.

## 7. Стек

- **Astro 4.16** — статическая генерация, `ViewTransitions`, content collections.
- TypeScript, обычный CSS ([src/styles/global.css](src/styles/global.css) +
  scoped-стили в компонентах), vanilla JS в компонентах. Внешних зависимостей,
  кроме Astro, нет.
- **Чат-бэкенд** — пока не делаем. Отдельный сервис (FastAPI), LLM-провайдер не
  финализирован; по личному правилу для пет-проектов — без Anthropic SDK.

## 8. Деплой — GitHub Pages **[изменилось]**

Не VPS/rsync/nginx, а **GitHub Pages через GitHub Actions**
([.github/workflows/deploy.yml](.github/workflows/deploy.yml), `withastro/action`):
push в `main` → сборка → деплой. Живёт на `https://konstantinkonov.github.io`
(`site` задан в [astro.config.mjs](astro.config.mjs)). Подробности, custom-домен
и нюанс с `base` для проектных сабпутей — в [DEPLOY.md](DEPLOY.md).

Локально: `npm run dev` (localhost:4321), `npm run build && npm run preview`.
